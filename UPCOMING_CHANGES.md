# Upcoming Changes

## Planned Features

- Optional embedding of `file_id.diz` during encoding, used as a display banner during extraction.
- Support for ANSI-encoded `file_id.diz`, with a self-contained runtime capable of displaying ANSI output safely in the terminal.
- Option to include a copy of the extractor inside the archive payload.
- Install argument to place the extractor in a user's PATH on extraction.
- Compliance with the ".grimoire" format for human- and AI-ingestible context, including timelined metadata describing how and why projects evolved.
- Continued positioning of BigShebang as a multi-disciplinary packaging tool for 0xacabOS.
